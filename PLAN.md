## Типы упражнений и правила реализации

### 1. Matching (Połącz / Dopasuj) — Соединение колонок
**Описание:** Соединить элементы левой колонки с элементами правой.  
**Примеры:** "Połącz pytania i odpowiedzi", "Dopasuj odpowiedzi do pytań", "Połącz kolumny", "Połącz słowa z obrazkami"

**Реализация:**
- Использовать класс `MatchingExercise` (click-to-connect)
- Левая колонка: вопросы/начала предложений (нумерация 0, 1, 2...)
- Правая колонка: ответы/окончания (буквы A, B, C...)
- Кликаем слева → кликаем справа = создаётся пара
- Пары выделяются одинаковым цветом
- Пример (0-й элемент) обычно заполнен заранее (prefilled)

**Если есть картинки:** Заменять описанием ситуации в скобках, если нет ответов в ключе — пропускать упражнение.

---

### 2. Fill-in (Uzupełnij) — Вставить слово/форму
**Описание:** Вставить правильную грамматическую форму в пропуск.  
**Примеры:** "Uzupełnij dialogi formami...", "Uzupełnij tekst zgodnie z podanym przykładem"

**Реализация:**
- `<input type="text" class="answer-input">` в месте пропуска
- Подсказка в placeholder: исходная форма в скобках, например `(być)₁`
- Пример (0-й) заполнен и заблокирован (`readonly`, зелёный фон)
- Контекст диалога/текста отображается вокруг
- Проверка: case-insensitive сравнение
- Если допускается несколько ответов — массив вариантов

**Варианты:**
- Из слов в рамке (word bank) → показывать `div.word-bank` сверху
- Из наведённой формы в скобках → показывать исходную форму рядом с полем
- Свободная трансформация → placeholder с подсказкой

---

### 3. Multiple choice (Zaznacz) — Выбор правильного варианта
**Описание:** Выбрать правильный вариант из нескольких предложенных.  
**Примеры:** "Zaznacz odpowiednie słowo", "Zaznacz poprawną odpowiedź"

**Реализация:**
- `<select class="answer-input">` с вариантами как `<option>`
- Или радио-кнопки для вопросов с 2-3 вариантами
- Первый option — пустой placeholder "—"
- При проверке: подсветка зелёным/красным

---

### 4. Odd-one-out (Które słowo nie pasuje?) — Найти лишнее
**Описание:** Указать слово, которое не подходит к остальным.  
**Примеры:** "Które słowo nie pasuje?", "Które słowo tu nie pasuje?"

**Реализация:**
- Список слов как кликабельные элементы или select
- Пользователь выбирает одно «лишнее» слово
- При проверке: правильное выделяется

---

### 5. Table completion (Uzupełnij tabelkę) — Заполнение таблицы
**Описание:** Вписать формы в ячейки таблицы (парадигмы склонения/спряжения).  
**Примеры:** "Uzupełnij tabelki odpowiednimi słowami", "Uzupełnij tabelki"

**Реализация:**
- HTML `<table class="fill-table">`
- Заголовки ячеек фиксированы
- Некоторые ячейки предзаполнены (класс `.given`)
- Остальные — `<input type="text" class="answer-input">`

---

### 6. Picture labeling (Podpisz obrazki) — Подписать картинки
**Описание:** Подписать изображения правильными словами/фразами.  
**Примеры:** "Podpisz obrazki zgodnie z podanym przykładem"

**Реализация:**
- Так как картинки не сохраняем, дать словесное описание ситуации
- Поля для ввода с описанием того, что изображено
- Если описание невозможно без картинки и нет ответа — пометка "требуется учебник"

---

### 7. Word search (Znajdź czasowniki) — Поиск слов
**Описание:** Найти слова в таблице букв.  
**Примеры:** "Znajdź 12 czasowników (poziomo → i pionowo ↓)"

**Реализация:**
- Сетка букв в таблице
- Кликабельные ячейки: выделение найденных слов
- Или упрощённо: ввод найденных слов в текстовые поля
- **Альтернатива (проще):** Список полей для ввода найденных слов

---

### 8. Ordering (Ułóż) — Расстановка в правильном порядке
**Описание:** Расставить элементы в хронологическом/логическом порядке.  
**Примеры:** "Poukładaj obrazki zgodnie z kolejnością"

**Реализация:**
- Перетаскивание (drag & drop) элементов или ввод номеров порядка
- Элементы перемешаны, нужно расставить

---

### 9. Free writing (Napisz / Odpowiedz) — Свободное письмо
**Описание:** Написать свой текст (эссе, ответ на вопрос).  
**Примеры:** "Napisz list", "Odpowiedz na pytania", "Napisz swoją interpretację"

**Реализация:**
- **НЕ РЕАЛИЗОВЫВАТЬ** как интерактивное упражнение (нет возможности автопроверки)
- Показать задание с пометкой: "Упражнение для самостоятельной работы, автопроверка невозможна"
- Опционально: textarea для записи без проверки

---

### 10. Listening (Posłuchaj piosenki) — Аудирование
**Описание:** Прослушать песню и выполнить задание.

**Реализация:**
- **НЕ РЕАЛИЗОВЫВАТЬ** (нет аудио)
- Пометка: "Требуется прослушивание аудио, см. учебник"
- Можно показать QR-код или название песни для самостоятельного поиска

## Общие правила

### Ответы
1. Если ответ есть в ключе (стр. 227–245) — использовать его
2. Если ответа нет в ключе, но можно вывести из грамматики — составить самостоятельно с пометкой: *"Ответы составлены на основе грамматических правил, а не из официального ключа"*
3. Если ответ невозможно определить (свободное письмо, аудирование) — пометка без проверки

### Навигация
- Каждое упражнение = отдельный HTML-файл
- Имя файла: `chX_secY_exZ.html` (глава_раздел_упражнение)
  - Пример: `ch1_0_ex1.html` — Глава I, раздел "Skąd jesteś?", упр. 1
- Кнопки: ← Wstecz | Menu | Dalej →
- Главная страница `index.html` — оглавление со ссылками на все упражнения, организованное по главам

### Внешний вид
- Общий CSS: `style.css`
- Общий JS: `script.js`
- Текст задания — на польском (как в учебнике)
- Интерфейс навигации/проверки — на русском (для пользователя)
- Пометки/предупреждения — на русском

### Обработка ошибок при проверке
- Правильно → зелёная подсветка
- Неправильно → красная подсветка + показать правильный ответ (`→ правильный`)
- Внизу: общий результат X/Y (Z%)

### Что НЕ реализуем
- Упражнения на свободное письмо (Napisz, Odpowiedz) — только показываем задание
- Аудио-упражнения (Posłuchaj) — только задание + ссылка
- Упражнения, которые невозможны без картинок И не имеют ответа в ключе

## Технические заметки

- Все файлы — статический HTML/CSS/JS, без серверной части
- Ответы хранятся в JavaScript на каждой странице (в клиентском коде)
- Пути к CSS/JS относительные: `../style.css`, `../script.js`
- Кодировка UTF-8 обязательна (польские символы: ą, ć, ę, ł, ń, ó, ś, ź, ż)
- Мобильная адаптивность через CSS media queries
